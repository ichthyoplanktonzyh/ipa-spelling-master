# IPA Spelling Master — 结构化系统分析文档

> 分析方法：数据流图 (Data Flow Diagram, DFD)  
> 建模工具：Yourdon/DeMarco 符号体系  
> 架构版本：v2 — 本地词库 + Web Speech API（纯前端架构）

---

## 1. 系统概述

### 1.1 系统目标
IPA Spelling Master 是一个纯前端 Web 应用，用于美式英语 IPA 音标拼写训练。核心业务流程为：系统从本地词库随机抽取训练题目 → 通过浏览器 Web Speech API 合成发音 → 用户听取发音 → 用户通过 IPA 键盘输入音标 → 系统判定答案并反馈结果 → 进入下一题直至完成全部训练。

### 1.2 分析范围
本分析覆盖系统的全部功能：词库管理、语音合成、用户交互输入、答案判定、进度追踪、难度切换。不包含未来规划的扩展功能（如用户账户系统、学习记录持久化等）。

### 1.3 架构特点
- **纯前端**：无服务端代码，全部逻辑在浏览器中运行
- **零外部 API**：发音使用浏览器内置 Web Speech API，词库存于本地
- **即时响应**：所有数据操作均为内存级别，无网络 I/O

### 1.4 分析方法说明
采用结构化分析方法，以 DFD 为核心建模工具，自顶向下逐层分解：
- **上下文图 (Context Diagram)**：系统与外部实体的数据交互全景
- **第 0 层 DFD (Level 0)**：系统顶层数据流，识别主要处理过程
- **第 1 层 DFD (Level 1)**：对 Level 0 的核心加工进行分解细化
- **数据字典**：定义所有数据流的组成
- **加工说明**：描述每个加工的输入、输出与处理逻辑

## 6. 加工说明 (Process Specification)

### P1.1 构建语音参数
```
输入：单词文本 (string)
输出：SpeechSynthesisUtterance 对象
处理逻辑：
  1. const utterance = new SpeechSynthesisUtterance(word)
  2. utterance.lang = 'en-US'
  3. utterance.rate = 0.9
  4. 可选：从 speechSynthesis.getVoices() 中筛选最佳 en-US 语音
  5. 传递至 P1.2
```

### P2.2 出题抽选
```
输入：难度等级 (Difficulty)
输出：WordData[] (10 个随机不重复单词)
处理逻辑：
  1. const pool = DS1[difficulty]  // 从词库取对应难度数组
  2. const shuffled = [...pool].sort(() => Math.random() - 0.5)  // Fisher-Yates 变体
  3. const selected = shuffled.slice(0, 10)
  4. 存入 DS3
  5. 同时重置 currentIndex=0, score=0, feedback='neutral', userInput=''
```

### P2.4 答案判定
```
输入：userInput (string), currentWord.ipa (string)
输出：判定结果 (correct/incorrect) + 错误时返回正确 IPA 和单词
处理逻辑：
  1. IF feedback != 'neutral' THEN 直接返回（防止重复判定）
  2. normalizedInput = userInput.trim().replace(/^\/|\/$/g, '')
  3. normalizedTarget = currentWord.ipa.trim().replace(/^\/|\/$/g, '')
  4. IF normalizedInput === normalizedTarget THEN
       feedback = 'correct', score += 1
     ELSE
       feedback = 'incorrect'
  5. 传递结果至 P2.5 和 P3.2
```

### P1.3 播放状态管理
```
输入：SpeechSynthesisUtterance 对象
输出：通过扬声器播放音频，返回播放状态
处理逻辑：
  1. speechSynthesis.cancel()  // 取消已在进行中的播放
  2. utterance.onstart = () => isPlaying = true
  3. utterance.onend   = () => isPlaying = false
  4. utterance.onerror = () => { isPlaying = false; 降级处理 }
  5. speechSynthesis.speak(utterance)
```

### P3.6 操作调度
```
输入：用户操作事件 (enum)
输出：路由至对应加工
处理逻辑：
  · 'play'    → 取 P2.3 当前单词文本 → P1.1
  · 'check'   → 取 P3.5 当前输入 → P2.4
  · 'next'    → P2.3 (currentIndex++)，重置输入和反馈
  · 'newSet'  → P2.2 (重新抽选)
  · 'changeDifficulty' → P2.1 → P2.2 (重新抽选)
  · 'toggleKeypad' → 切换 showKeypad 状态 → P3.1 重渲染
  · 'insertChar' → P3.5 追加字符
  · 'deleteChar' → P3.5 删除末字符
```

## 8. 与 v1 (Gemini 架构) 的差异对比

| 维度 | v1 (Gemini) | v2 (本地 + Web Speech) |
|------|------------|----------------------|
| 外部实体 | Gemini AI Service (2 个 API) | Web Speech API (浏览器内置) |
| 数据存储 | 无本地持久化，依赖 API 响应 | DS1 本地词库 (80 词/级) |
| 词库来源 | 每次 API 调用动态生成 | 静态 TS 文件，编译时内联 |
| 音频来源 | Gemini TTS (Base64 → Web Audio API) | SpeechSynthesis (OS 原生) |
| 网络依赖 | 必须联网，依赖 API 可用性 | 加载后完全离线可用 |
| 延迟特性 | 词库 2-5s，TTS 2-3s | 词库 < 1ms，TTS < 200ms |
| 后端需求 | Express 服务代理 API Key | 无后端，纯静态文件 |
| 费用 | Gemini API 按量计费 | 零费用 |
| 加工数量 | 17 个子加工 | 14 个子加工 |
| 可靠性 | 受 API 限流、网络中断影响 | 仅受浏览器兼容性影响 |
