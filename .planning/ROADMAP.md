# 多语言语音训练平台 Roadmap

## 1. 路线图目标

本路线图将 PhoneticMaster 从英语单语 IPA 工具演进为轻量、纯前端的多语言语音感知训练器。阶段策略是：

1. 先完成 **不依赖诊断的独立训练器**。
2. 再补齐 **专项训练与最小对立体听辨**。
3. 然后加入 **本地个性化与 L1-aware 推荐**。
4. 再补齐 **可信内容、标准音频、引导式学习和多样练习**，并用新语言验证扩展边界。
5. 最后按产品验证结果决定是否进入云端增强。

核心原则：训练器本身必须完整；推荐和诊断是加速器，不是入口门槛。

## 3. 技术选择与决策门

| 决策点 | 选项 | 决策 | 理由 |
|--------|------|------|------|
| UI 框架 | React / Vue / Svelte | React | 项目已有 React 代码库 |
| 样式方案 | CSS Modules / Tailwind / styled | Tailwind | 已有 Tailwind 基础设施 |
| 播放来源 | Web Speech API / 标准音频 / 后端 TTS | 标准音频可选 + Web Speech fallback | M5 从许可明确的小规模高价值内容开始，保持零后端依赖 |
| 状态管理 | useState / Zustand / Jotai | useState | 当前规模足够，不需要额外库 |
| 判定方式 | 字符串比较 / 音素级比较 | 音素级 (phonemeJudge) | 支持 nearMatch 和 diff 反馈 |
| 训练历史 | localStorage / IndexedDB / 后端 DB | localStorage 起步 | MVP 轻量、隐私压力低、可静态发布 |
| 用户录音 | localStorage / IndexedDB / 云存储 | IndexedDB（M5 规划） | 二进制录音不适合 localStorage；默认只保存在设备本地 |
| L1 数据来源 | 硬编码 / 远程 API / 社区编辑 | 硬编码 | v1 可控，云端增强阶段再考虑社区贡献 |
| 后端 | 必需 / 可选 | 可选远期 | MVP 的训练、推荐、记录都可在浏览器内完成 |

## 5. M0 — 原始英语 IPA 工具

> 状态：✅ 已完成。项目起点。

- **覆盖需求**：CORE-001 ~ CORE-005

  - M0.1 IPA 解析器 + 音素分词
  - M0.2 英语音标键盘 + 拼写判定
  - M0.3 Web Speech API TTS
  - M0.4 难度分级 + 词汇筛选
- **交付物**：可运行的英语 IPA 训练 SPA
- **退出条件**：听音 → 输入 IPA → 判定 完整流程可用

## 6. M1 — 多语言架构 + 汉语 + L1 推荐

> 状态：✅ 已完成。2026-06-22。

- **目标**：将英语单语工具扩展为 LanguageProfile-driven 多语言平台，新增汉语拼音训练和 L1-aware 智能推荐基础层
- **覆盖需求**：ARCH-001 ~ ARCH-004, LANG-001 ~ LANG-003, L1-001 ~ L1-003
- **主要任务**：
  - M1.1 类型系统重构 (TrainingItem, LanguageProfile, JudgeResult)
  - M1.2 汉语 Profile (声母/韵母/声调 + 拼音解析 + 词库)
  - M1.3 L1×L2 难度映射 (zh→en, en→zh, difficultyMap API)
  - M1.4 UI 重构 (PhoneticKeypad, OnboardingView, SmartRecommend, App.tsx)
  - M1.5 编译验证与修复
- **交付物**：
  - Profile 注册表 + 双语言 profile
  - L1-aware 推荐面板基础版
  - Onboarding 引导流程
  - 零错误编译 + 生产构建
- **退出条件**：
  - ✅ `npx tsc --noEmit` 零错误
  - ✅ `npm run build` 成功
  - ✅ 英语/汉语双语言训练流程可用
  - ✅ 新语言接入仅需 profile + 词库，无框架改动

## 7. M2 — 纯前端独立训练器 MVP

> 状态：✅ 已完成。2026-07-02。

- **目标**：让用户无需 L1 诊断、无需账号、无需后端，也能独立完成一次有效训练会话，并可直接发布到公网。
- **覆盖需求**：CORE-001 ~ CORE-005, ARCH-004, TRAIN-CORE-001 ~ TRAIN-CORE-005, FEED-001 ~ FEED-002, DATA-LOCAL-001, PLAT-STATIC-001
- **主要 phase**：
  - ✅ 2.1 Standalone Training Core：独立训练入口、训练类型选择、目标语言/难度/主题选择（2026-07-02 完成）
  - ✅ 2.2 Data Cleaning：清洗/校验词库、Profile 与 L1×L2 映射数据（2026-07-02 完成）
  - ✅ 2.3 Feedback & Session Results：nearMatch/diff 可视化、会话结果页、本地最近训练记录（2026-07-02 完成）
  - ✅ 2.4 Publish Readiness：静态部署检查、README/PRD 对齐、基础手动 QA 清单（2026-07-02 完成）
- **交付物**：
  - 不依赖 L1 的训练入口
  - 听音拼写 + 看词听音两种完整训练会话
  - nearMatch 与错误音素差异反馈
  - 会话结果页和本地最近训练记录
  - 纯前端公网发布说明
- **退出条件**：
  - 用户跳过/不选择 L1 时仍可训练
  - 一轮训练具备开始、进度、判定、结果、再练完整闭环
  - `npx tsc --noEmit` 和 `npm run build` 通过
  - 静态托管方案清晰，不需要后端服务
- **建议投入**：3-5 天

## 8. M3 — 专项训练 + 最小对立体

> 状态：✅ 已完成。Phase 3.1、3.2 已完成（2026-07-02），Phase 3.3 已完成（2026-07-03）。

- **目标**：把训练器从“通用练习”升级为“针对具体声音问题的专项练习”。
- **覆盖需求**：TRAIN-001 ~ TRAIN-003, FEED-003
- **主要 phase**：
  - ✅ 3.1 Targeted Minimal Pairs：最小对立体 A/B 听辨训练（2026-07-02 完成）
  - ✅ 3.2 Chinese Structured Input：声母 → 韵母 → 声调三步输入（2026-07-02 完成）
  - ✅ 3.3 Phoneme Detail Panel：音素详情、PAM/SLM 原因、例词和专项入口（2026-07-03 完成）
- **交付物**：
  - 最小对立体听辨模式
  - 音素/声调/韵母专项训练入口
  - 中文拼音三步输入原型
  - 音素详情面板
- **退出条件**：
  - 用户可手动选择某个音素/声调进入专项训练
  - minimalPairs 数据能生成可完成、可计分的练习
  - 中文拼音输入不再只是简单追加字符
- **建议投入**：4-6 天

## 9. M4 — 本地个性化 + L1-aware 教练

> 状态：✅ Phase 4.1 已完成（2026-07-03）。原 4.2/4.3 的 MVP 范围已并入 4.1。

- **目标**：让推荐从静态语言学建议变成“理论难点 + 用户历史表现”的本地教练。
- **覆盖需求**：L1-001 ~ L1-003, DATA-001 ~ DATA-003
- **主要 phase**：
  - ✅ 4.1 Local Personalization：训练历史、音素掌握度、本地推荐排序、训练完成后的下一步建议、可选 Coach 面板（2026-07-03 完成）
  - ✅ 4.2 Coach Recommendations：MVP 范围并入 4.1
  - ✅ 4.3 Smart Panel Redesign：MVP 范围并入 4.1
- **交付物**：
  - localStorage 存储抽象
  - 按 (L2, topic, phoneme?) 聚合的掌握度
  - 推荐排序融合 L1 难度和历史正确率
  - 训练完成后的“下一轮建议”
- **退出条件**：
  - ✅ 推荐顺序会随用户训练表现变化
  - ✅ 用户不使用 L1 推荐也不影响基础训练
  - ✅ 本地数据读写失败时训练仍可用
- **建议投入**：3-5 天

## 10. M5 — 可信内容与练习体验扩展

> 状态：🧭 已规划，待从 Phase 5.1 开始。

- **目标**：借鉴 iSpeakerReact 的内容体验优势，建立来源可追溯的学习内容、可信音频、引导式音素课程、本地录音回放和数据驱动练习，再用第 3 种语言验证扩展边界。
- **覆盖需求**：ARCH-005, LANG-004, CONTENT-001 ~ CONTENT-002, AUDIO-001 ~ AUDIO-002, PRACTICE-001 ~ PRACTICE-002
- **主要 phase**：
  - 🧭 5.1 Content Foundation：内容契约、来源/授权元数据、查询接口与数据校验
  - 🧭 5.2 Trusted Audio & Accent Variants：标准音频优先、TTS fallback、语言自定义口音变体
  - 🧭 5.3 Guided Phoneme Lessons：示范、上下文例词、重音/音节和专项入口的短学习路径
  - 🧭 5.4 Local Record & Review：MediaRecorder + IndexedDB 本地录音、回放与删除，不做机器评分
  - 🧭 5.5 Data-driven Exercise Templates：共享练习会话模型和至少两种新感知练习
  - 🧭 5.6 New Language Pilot：基于内容授权与维护成本选型并接入第 3 种语言
- **交付物**：
  - 语言无关的内容与媒体契约、来源校验门禁
  - 标准音频优先/TTS 降级的统一播放层
  - 可完成的音素短课程和本地录音复习
  - 至少两种新的数据驱动感知练习
  - 至少 1 种新语言 profile + 词库 + 专项内容
- **退出条件**：
  - 未确认授权的第三方课程和媒体不进入生产构建
  - 缺少扩展内容、标准音频、录音能力时基础训练可正常降级
  - 新内容和练习不在组件中引入语言特定分支
  - 新语言无需修改通用训练 UI 即可训练
- **明确不做**：
  - 复制 iSpeakerReact/Oxford 的课程、词表、音频或视频资产
  - Electron/Python/wav2vec2 发音评分
  - 一次性复刻 iSpeakerReact 的全部八种练习
- **建议投入**：按 phase 独立评估，优先完成 5.1-5.3 后再决定 5.4-5.6 节奏

## 11. M6 — 云端增强（可选）

> 状态：🧭 可选远期。只有当产品验证后明确需要时进入。

- **目标**：在保持纯前端核心可用的前提下，按需增加云端能力。
- **触发条件**：
  - 用户明确需要跨设备同步
  - Web Speech API 无法满足核心听辨质量
  - 需要账号、会员、课程进度或付费功能
  - 需要 ASR/Whisper 做产出训练
  - 需要社区贡献内容与审核流程
- **可能任务**：
  - M6.1 账号与同步
  - M6.2 标准音频托管或 TTS 服务
  - M6.3 ASR 发音评分
  - M6.4 内容管理与社区贡献
  - M6.5 产品分析与匿名埋点
- **退出原则**：
  - 云端能力不能破坏“打开即练”的纯前端体验
  - 本地训练仍应在无登录状态下可用

## 13. 风险登记册

| # | 风险 | 可能性 | 影响 | 缓解措施 |
|---|------|--------|------|----------|
| 1 | Web Speech API TTS 质量低 | 高 | 中 | 提供语音选择器；核心 minimal pairs 未来可托管标准音频 |
| 2 | L1×L2 映射数据专业性强，难以维护 | 中 | 高 | 先作为可选教练层；后续 JSON Schema + 审校流程 |
| 3 | 拼音三步输入模式复杂度高 | 中 | 中 | M3 先做原型验证 |
| 4 | 汉语词库不够丰富 | 中 | 中 | 预留 CEDICT/HSK 导入脚本接口 |
| 5 | 移动端键盘布局空间不足 | 高 | 中 | M2 保证不严重破版，完整移动端放后续 |
| 6 | 过早引入后端导致发布变慢 | 中 | 高 | M2-M4 坚持纯前端，后端进入 M6 可选阶段 |
| 7 | README/PRD 与新定位不一致 | 已缓解 | 中 | Phase 2.4 已统一对外文档 |
| 8 | 参考项目的代码许可证被误当作内容/媒体授权 | 中 | 高 | Phase 5.1 强制记录来源、许可证和可再分发状态；未确认内容不进生产包 |
| 9 | 标准音频和录音能力推高静态包体与浏览器复杂度 | 中 | 中 | 小规模 fixture 起步；标准音频可选；录音使用 IndexedDB 并提供完整降级 |

## 14. 质量门禁

| 门禁 | 条件 | 适用里程碑 |
|------|------|------------|
| 编译 | `npx tsc --noEmit` 零错误 | 全部 |
| 构建 | `npm run build` 成功 | 全部 |
| 独立训练 | 无 L1 诊断也能完成训练会话 | M2+ |
| 静态发布 | 无后端依赖即可部署 | M2+ |
| 架构 | 新语言接入无需修改框架代码 | M1+ |
| L1 覆盖 | 每个新语言至少 1 个 L1 映射 | M5+ |
| 词库 | 每个难度档至少 20 条目 | M5+ |
| 内容来源 | 外部内容具备来源、许可证与可再分发状态 | M5+ |
| 媒体降级 | 标准音频失败时不阻断训练，能回退 TTS 或无音频状态 | M5+ |
