# Phase 2.2 上下文

## 来源

Phase 2.1 完成后，独立训练入口已经成立。进入反馈和结果页前，需要先回答一个更上游的问题：哪些功能真正服务于 MVP 的语音感知训练闭环，哪些只是看起来高级但会拖慢核心验证。

本阶段来自 2026-07-02 的产品讨论：

- 必要需求围绕“选择 L2 → 开始一轮训练 → 得到可学习反馈 → 看到结果 → 再练/换主题”
- L1-aware 推荐是加速器，不是入口门槛
- 账号、后端、ASR、社交、课程体系、过早多语言扩张等应视为远期增强或伪需求

## 关键决策

- 先做 scope triage，再实现 Feedback & Session Results。
- Phase 2.2 是产品/架构澄清阶段，不写业务代码。
- 原 Phase 2.2 Feedback & Session Results 顺延为 Phase 2.3。
- MVP 必要功能必须直接服务训练闭环。
- DDD 上，MVP 核心域仍是 Training Core + Feedback。

## 约束

- 需求文档中的稳定 ID 不重命名。
- 不用“删除”掩盖远期需求；应该将远期增强放回 M3/M4/M5/M6。
- 伪需求需要明确说明为什么不服务当前核心闭环。
- 本阶段结束后，Phase 2.3 应能直接按裁剪后的范围实现结果页和本地最近记录。
