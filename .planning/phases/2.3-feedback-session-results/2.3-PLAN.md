# Phase 2.3: Feedback & Session Results

## 目标

把训练反馈从“对/错提示”升级为“可学习的错误反馈”，并用结果页替代简单 alert，形成完整会话闭环。

## 任务

- [ ] 2.3.1 为 feedback 增加 nearMatch 独立状态，避免简单按 correct 计分
- [ ] 2.3.2 使用 JudgeResult.diffs 展示 expected vs actual 差异
- [ ] 2.3.3 设计会话结果页/面板，显示得分、准确率、错题和再练入口
- [ ] 2.3.4 记录最近训练会话到本地存储
- [ ] 2.3.5 增加清除本地训练记录能力
- [ ] 2.3.6 为 judge/parser/storage 增加最小单元测试或手动 QA 清单

## 退出条件

- [ ] nearMatch 显示为“几乎正确”并展示差异位置
- [ ] incorrect 显示具体错误音素/声母/韵母/声调
- [ ] 完成本轮训练后显示结果页，不再只使用 alert
- [ ] 最近训练记录写入失败时应用不崩溃
- [ ] `npx tsc --noEmit` 通过
- [ ] `npm run build` 通过

## 影响文件

- `src/App.tsx`
- `src/components/SessionResultView.tsx`（新增）
- `src/components/PhonemeDiffView.tsx`（新增）
- `src/utils/judge.ts`
- `src/utils/storage.ts`（新增）
- `src/types.ts`

## 约束

- 本阶段只做本地最近记录，不做云端同步
- diff 反馈必须通过 LanguageProfile.judge 的 JudgeResult 驱动
- 具体 scope 以前置 Phase 2.2 MVP Scope Triage 的分类结果为准
