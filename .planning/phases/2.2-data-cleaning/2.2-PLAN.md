# Phase 2.2: Data Cleaning

## 目标

在进入结果页和训练历史前，清洗并校验训练数据、语言 Profile 数据和 L1×L2 映射数据，让后续反馈、结果统计和推荐建立在可信数据上。

## 任务

- [x] 2.2.1 迁移英语词库：将 `src/data/wordBank.ts` 从 legacy `WordData` 格式迁移为 `TrainingItem[]`
- [x] 2.2.2 移除英语 profile 的运行时转换逻辑，确保 `englishProfile.wordBank` 直接使用统一数据格式
- [x] 2.2.3 建立数据校验脚本或工具：检查词库字段完整性、重复 identity、difficulty tier 一致性
- [x] 2.2.4 校验发音记号：所有 `TrainingItem.pronunciation` 必须能被对应 `LanguageProfile.parseNotation` 解析
- [x] 2.2.5 校验音素引用：词库解析出的 token、`keypadLayout` 和 L1 difficulty hardPhonemes 必须引用已声明的 profile phoneme
- [x] 2.2.6 校验中文拼音数据：`pronunciation` 使用数字声调 canonical form，`pronunciationAlt` 仅作为显示形式
- [x] 2.2.7 校验 L1×L2 映射：level 范围、l1/l2 code、hardFeatures、minimalPairs 基础格式
- [x] 2.2.8 更新 `DATA-MODEL.md`、`CONCERNS.md`、`TESTING.md`，记录数据清洗规则和剩余缺口

## 退出条件

- [x] 新代码不再依赖 legacy `WordData` 作为英语词库运行格式
- [x] 词库、Profile、L1 映射至少有一个可重复运行的数据校验入口
- [x] 英语和中文词库通过字段、identity、difficulty、notation、phoneme reference 校验
- [x] L1×L2 映射引用的音素和特征均能在对应 profile 中找到
- [x] `npx tsc --noEmit` 通过
- [x] `npm run build` 通过

## 约束

- 不改变用户训练流程和 UI 行为
- 不引入后端、数据库或远程数据源
- 不在本阶段扩充新语言或新词库规模
- 不删除可用数据；清洗时应优先规范格式、补充字段、标记问题
- 校验逻辑应可在本地重复运行，便于后续新增语言/词库复用
