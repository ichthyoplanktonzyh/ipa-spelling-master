# Phase 5.1 上下文

## 来源

M4 完成后，项目原计划直接进入第 3 种目标语言接入。2026-07-15 对
[iSpeakerReact](https://github.com/learnercraft/ispeakerreact) 的产品与源码调研表明，当前更值得优先吸收的是：

- 结构化的音素学习内容，而不是继续堆叠训练入口；
- 标准音频、口音变体和 TTS 降级；
- 示范 → 例词 → 练习 → 复习的学习路径；
- 数据驱动的多种听辨练习；
- 浏览器本地录音与回放，而非立即做 ASR 发音评分。

## 关键决策

- 将 M5 调整为“可信内容与练习体验扩展”，新语言试点保留在 M5 后段。
- 借鉴 iSpeakerReact 的产品机制和通用代码思路，不复制其 Oxford 词表、课程文本、音频或视频资产。
- 所有外部内容必须记录来源、许可证和可再分发状态；状态不明确时不得进入生产包。
- 内容能力继续通过 `LanguageProfile` 或 profile-facing 查询层提供，组件不直接依赖具体语言数据文件。
- Phase 5.1 先建立内容契约、来源规则和校验门禁，不在本阶段引入录音、ASR 或大型媒体包。

## 约束

- 保持静态部署和无账号使用。
- 缺少扩展内容时，现有训练器、音素详情和 TTS 必须继续工作。
- 新类型仍以 `src/types.ts` 为共享类型唯一来源。
- 新内容数据必须进入 `npm run validate:data` 的校验范围。
- 不修改已经冻结的 M0-M4 phase 文件。
