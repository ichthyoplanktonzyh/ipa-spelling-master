# Phase 5.1: Content Foundation

## 目标

建立可验证、可追溯、语言无关的学习内容契约，为标准音频、音素课程、录音和新练习提供稳定基础。

## 任务

- [ ] 5.1.1 盘点现有 `TrainingItem`、`PhonemeDetail`、minimal pair 与音频字段的重叠和缺口
- [ ] 5.1.2 定义学习内容、媒体来源和授权元数据的共享类型
- [ ] 5.1.3 明确 profile-facing 内容查询接口，避免组件直接导入 `src/data/*`
- [ ] 5.1.4 为英语和中文各准备一小组自有或许可明确的 fixture 内容
- [ ] 5.1.5 扩展 `npm run validate:data`，校验引用、来源、许可证状态和可选字段降级
- [ ] 5.1.6 为内容查询与数据校验补充 fixture-based 测试
- [ ] 5.1.7 更新架构、结构、数据模型、约定、测试和风险文档

## 退出条件

- [ ] 每条外部学习内容都能追溯来源与可再分发状态
- [ ] 未确认授权的内容不会进入生产构建
- [ ] 英语和中文能通过同一内容接口读取 fixture
- [ ] 缺少扩展内容的 profile 能无错误降级到现有体验
- [ ] `npm run validate:data`、`npm run lint`、`npm run test` 和 `npm run build` 通过

## 影响文件

- `src/types.ts`
- `src/profiles/*`
- `src/data/*`
- `src/utils/phonemeDetails.ts`
- `scripts/validateData.ts`
- `src/utils/__tests__/*`
- `.planning/codebase/*`

## 非目标

- 引入 iSpeakerReact/Oxford 的课程或媒体资产
- 下载或托管大规模音视频包
- 录音、ASR 或发音评分
- 新增目标语言
