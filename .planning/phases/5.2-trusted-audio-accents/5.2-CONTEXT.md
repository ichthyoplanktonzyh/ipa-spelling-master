# Phase 5.2 上下文

## 来源

现有普通训练和 minimal pair 主要依赖 Web Speech API。Phase 3.1 QA 已确认 TTS 对精细听辨只能“勉强可用”。iSpeakerReact 的可借鉴点是优先播放策划过的标准音频，并把口音作为显式内容维度。

## 关键决策

- 播放顺序为：许可明确的标准音频 → 可用的 profile TTS → 可恢复的无音频状态。
- 口音是可选媒体变体，不改变 `TrainingItem` 的身份和正确答案。
- 音频选择逻辑集中在 profile-facing service，不散落在组件中。
- 首批只覆盖少量高价值英语 minimal pairs/音素 fixture；未覆盖内容继续使用 TTS。

## 约束

- 静态资源体积必须可控。
- 不能因某个音频 URL 失败而阻断训练会话。
- 中文与未来语言不得被迫采用英语的 British/American 枚举。
