# 多语言语音训练平台 Requirements

## 1. 文档用途

定义 PhoneticMaster 平台的需求条目，每条需求具有稳定 ID、优先级和验收标准。

## 2. 范围与术语

### 优先级定义

| 级别 | 含义 |
|------|------|
| P0 | 必须满足，缺失则产品不可用 |
| P1 | 重要功能，缺失则体验明显降级 |
| P2 | 锦上添花，可延后 |

### 术语

| 术语 | 含义 |
|------|------|
| L1 | 学习者的母语 (Native Language) |
| L2 | 学习者的目标语言 (Target Language) |
| Profile | LanguageProfile，一个语言的完整声明 |
| 音素 | 语音的最小区分单位，包括元音、辅音、声母、韵母、声调 |
| nearMatch | 大部分音素正确但有少量差异的判定结果 |
| PAM | 感知同化模型 (Perceptual Assimilation Model) |
| SLM | 语音学习模型 (Speech Learning Model) |

## 4. CORE — 核心训练

### REQ CORE-001: 听音拼写

- 优先级：P0
- 阶段：M0
- 需求：用户听到 TTS 发音后，使用屏幕键盘或物理键盘输入音标/拼音符号
- 验收标准：
  - 点击播放按钮可听到单词发音
  - 可通过屏幕键盘逐符号输入
  - 可通过物理键盘直接输入
- 依赖：—

### REQ CORE-002: 拼写判定

- 优先级：P0
- 阶段：M0
- 需求：系统判定用户输入是否正确，支持精确匹配和近似匹配
- 验收标准：
  - 完全正确显示绿色反馈
  - 完全错误显示红色反馈 + 正确答案
  - nearMatch（1 个音素差异）显示特殊提示
- 依赖：ARCH-001

### REQ CORE-003: 难度分级

- 优先级：P1
- 阶段：M0
- 需求：词库按频率/难度分为基础/进阶/挑战三档
- 验收标准：
  - 用户可在 header 切换难度
  - 切换后立即刷新词集
- 依赖：—

### REQ CORE-004: 音素筛选

- 优先级：P1
- 阶段：M0
- 需求：用户可按特定音素筛选包含该音素的单词
- 验收标准：
  - 下拉菜单列出所有音素及其匹配词数
  - 选择后仅显示包含该音素的词汇
- 依赖：ARCH-002

### REQ CORE-005: 词汇量控制

- 优先级：P2
- 阶段：M0
- 需求：用户可设置每轮训练的单词数量 (1-50)
- 验收标准：
  - 数字输入框可调节
  - 切换后立即刷新词集
- 依赖：—

## 5. ARCH — 架构

### REQ ARCH-001: LanguageProfile 接口

- 优先级：P0
- 阶段：M1
- 需求：每个目标语言声明为一个 LanguageProfile 对象，包含音素清单、键盘布局、解析器、判定器、词库等全部语言特定逻辑
- 验收标准：
  - UI 代码不含语言特定 if/else
  - 所有语言行为通过 profile 属性驱动
- 依赖：—

### REQ ARCH-002: Profile 注册表

- 优先级：P0
- 阶段：M1
- 需求：提供 getProfile(code) 查询和 getAllProfiles() 枚举 API
- 验收标准：
  - 新语言只需 import profile + 加入 SUPPORTED_L2 数组
  - O(1) code 查询
- 依赖：ARCH-001

### REQ ARCH-003: 音素级判定

- 优先级：P0
- 阶段：M1
- 需求：判定逻辑基于音素数组比较而非字符串相等
- 验收标准：
  - phonemeJudge(input, target, parseFn) 返回 JudgeResult
  - JudgeResult 包含 correct, nearMatch, diffs
  - 支持 per-position 差异
- 依赖：ARCH-001

### REQ ARCH-004: 双模式训练

- 优先级：P1
- 阶段：M1
- 需求：支持拼写模式（输入判定）和训练模式（浏览记忆）两种方式
- 验收标准：
  - 可通过 header 切换模式
  - 训练模式支持键盘导航 (Space/←/→)
- 依赖：—

### REQ ARCH-005: 零框架改动扩展

- 优先级：P0
- 阶段：M5
- 需求：添加新语言时不需要修改任何框架/UI 代码
- 验收标准：
  - 新语言 = 1 个 profile 文件 + 1 个词库文件 + 1 行 import
  - `npx tsc --noEmit` 和 `npm run build` 不需要额外修改即可通过
- 依赖：ARCH-001, ARCH-002

## 6. TRAIN-CORE — 独立训练器

### REQ TRAIN-CORE-001: 无诊断训练入口

- 优先级：P0
- 阶段：M2
- 需求：用户不选择 L1、不进入推荐面板，也能选择目标语言和训练模式并开始训练
- 验收标准：
  - 首次访问可直接进入训练器或跳过 L1 诊断
  - 至少可选择 L2、训练模式、难度
  - L1 为空时不阻塞训练流程
- 依赖：ARCH-001, ARCH-002

### REQ TRAIN-CORE-002: 训练类型选择

- 优先级：P0
- 阶段：M2
- 需求：训练器提供清晰的训练类型入口，至少包含听音拼写和看词听音
- 验收标准：
  - 用户可在同一轮开始前选择训练类型
  - 切换训练类型会重置当前会话状态
  - 训练类型不依赖具体语言硬编码
- 依赖：ARCH-004

### REQ TRAIN-CORE-003: 完整训练会话

- 优先级：P0
- 阶段：M2
- 需求：每轮训练具有开始、进度、作答/浏览、反馈、完成结果、再练一次的闭环
- 验收标准：
  - 一轮训练包含固定数量题目
  - 页面显示当前进度和本轮总数
  - 完成后显示结果，而不是只弹出 alert
  - 用户可基于当前设置重新开始一轮
- 依赖：CORE-001, CORE-002

### REQ TRAIN-CORE-004: 主题选择

- 优先级：P1
- 阶段：M2
- 需求：用户可手动选择训练主题，如全部、特定音素、特定声调/韵母
- 验收标准：
  - 主题选择来自 LanguageProfile / phonemeGroups
  - 选择主题后训练题目只来自匹配词集
  - 空词集时显示可恢复的空状态
- 依赖：CORE-004

### REQ TRAIN-CORE-005: 纯前端可运行

- 优先级：P0
- 阶段：M2
- 需求：独立训练器不依赖后端服务、数据库或账号体系
- 验收标准：
  - 断开后端服务时，静态构建产物仍可完成训练
  - 训练数据来自前端包内词库
  - 用户状态仅使用浏览器本地能力保存
- 依赖：PLAT-STATIC-001

## 7. LANG — 语言扩展

### REQ LANG-001: 英语 IPA Profile

- 优先级：P0
- 阶段：M0
- 需求：英语美式 IPA 训练的完整 profile
- 验收标准：
  - 16 元音 + 24 辅音 + 4 标记
  - tokenizeIpa 解析器
  - 英语词库 (basic/intermediate/advanced)
- 依赖：ARCH-001

### REQ LANG-002: 汉语拼音 Profile

- 优先级：P0
- 阶段：M1
- 需求：汉语拼音训练的完整 profile
- 验收标准：
  - 21 声母 + 35+ 韵母 + 5 声调
  - 拼音解析器（声调符号 + 数字声调双支持）
  - HSK 1-3 词库（≥150 条目）
  - 训练模式显示声调符号，拼写模式接受数字声调
- 依赖：ARCH-001

### REQ LANG-003: 语言选择

- 优先级：P0
- 阶段：M1
- 需求：用户可选择目标语言 (L2)，并可选选择母语 (L1) 以启用个性化推荐；L1 不应成为训练入口的硬阻塞
- 验收标准：
  - 用户至少能确定一个 L2 并进入训练
  - 显示可选 L1 列表（≥10 种语言）
  - 未选择 L1 时训练可用，推荐面板显示降级提示或隐藏
  - 显示 L2 列表（来自 getAllProfiles()）
  - L1 === L2 时提示重新选择
  - 选择结果持久化到 localStorage
- 依赖：ARCH-002

### REQ LANG-004: 新语言接入

- 优先级：P1
- 阶段：M5
- 需求：支持接入日语、西语等第 3/4 种语言
- 验收标准：
  - 每种新语言仅需 profile + 词库文件
  - 自动出现在 Onboarding 和 header 选择器中
- 依赖：ARCH-005

## 8. L1 — L1 感知

### REQ L1-001: L1×L2 难度映射

- 优先级：P0
- 阶段：M1
- 需求：为每对 (L1, L2) 提供音素难度级别 (1-5) 和原因
- 验收标准：
  - 每个 hardPhoneme 包含 level, reason, l1Equivalence, minimalPairs
  - 每个 hardFeature 包含 level, reason
  - 无映射时返回 level 0 降级
- 依赖：ARCH-001

### REQ L1-002: 智能推荐面板

- 优先级：P1
- 阶段：M1
- 需求：作为可选智能层显示 L1→L2 的难点音素和特征，不阻塞独立训练器使用
- 验收标准：
  - 展示 top N 难点音素（星级 + 原因）
  - 展示 top M 难点特征（星级 + 原因）
  - 点击音素可启动该音素的训练
  - 无 L1 或无映射时显示提示
- 依赖：L1-001

### REQ L1-003: L1×L2 映射数据

- 优先级：P1
- 阶段：M1
- 需求：提供 zh→en 和 en→zh 两组映射数据
- 验收标准：
  - zh→en: ≥8 个 hardPhonemes + ≥4 个 hardFeatures
  - en→zh: ≥8 个 hardPhonemes + ≥3 个 hardFeatures
  - 每个 hardPhoneme 有 PAM/SLM 理论依据
- 依赖：L1-001

## 9. FEED — 反馈增强

### REQ FEED-001: 音素差异可视化

- 优先级：P1
- 阶段：M2
- 需求：判定错误时高亮显示哪个音素与预期不符
- 验收标准：
  - 利用 JudgeResult.diffs 定位错误位置
  - 显示 expected vs actual 对比
- 依赖：ARCH-003

### REQ FEED-002: nearMatch 提示

- 优先级：P1
- 阶段：M2
- 需求：nearMatch 时显示"几乎正确"的温和反馈
- 验收标准：
  - 显示 "Almost Correct" 而非 "Incorrect"
  - 同时显示差异位置
- 依赖：ARCH-003, FEED-001

### REQ FEED-003: 音素详情弹窗

- 优先级：P2
- 阶段：M3
- 需求：点击音素符号可查看详细信息
- 验收标准：
  - 显示 PAM/SLM 原因
  - 显示 L1 等价音
  - 显示最小对立体例词
- 依赖：L1-001

## 10. TRAIN — 专项训练

### REQ TRAIN-001: 最小对立体训练

- 优先级：P1
- 阶段：M3
- 需求：基于 L1L2Difficulty.minimalPairs 生成对比听辨练习
- 验收标准：
  - 播放两个最小对立体单词
  - 用户选择听到的是哪个
  - 统计正确率
- 依赖：L1-001

### REQ TRAIN-002: 汉语三步输入

- 优先级：P1
- 阶段：M3
- 需求：汉语拼写模式分为声母→韵母→声调三步
- 验收标准：
  - 第 1 步只显示声母键盘
  - 选完声母后切换到韵母键盘
  - 选完韵母后切换到声调键盘
  - 自动拼接为完整拼音
- 依赖：LANG-002

### REQ TRAIN-003: 跟读提示

- 优先级：P2
- 阶段：M3
- 需求：训练模式显示释义和跟读提示
- 验收标准：
  - 显示 TrainingItem.definition
  - 显示"试试跟读"提示
- 依赖：—

## 11. DATA-LOCAL — 本地训练数据

### REQ DATA-LOCAL-001: 最近训练记录

- 优先级：P1
- 阶段：M2
- 需求：MVP 在本地记录最近训练会话结果，用于结果页回顾和后续个性化的基础数据
- 验收标准：
  - 记录训练时间、L2、训练类型、主题、题目数、正确数
  - localStorage 不可用时训练不崩溃
  - 用户可以清除本地记录
- 依赖：TRAIN-CORE-003

### REQ DATA-LOCAL-002: 存储抽象

- 优先级：P1
- 阶段：M2
- 需求：本地存储读写通过独立模块完成，避免组件直接散落 localStorage 逻辑
- 验收标准：
  - 提供读取、写入、清除训练记录的工具函数
  - 所有浏览器存储异常都有 try/catch 降级
  - 未来可替换为 IndexedDB 或后端 API
- 依赖：—

## 12. DATA — 训练数据与个性化

### REQ DATA-001: 训练历史记录

- 优先级：P1
- 阶段：M4
- 需求：记录每次训练的结果到 localStorage
- 验收标准：
  - 记录音素级别的对/错
  - 按 (L1, L2) 分组存储
- 依赖：—

### REQ DATA-002: 音素掌握度

- 优先级：P1
- 阶段：M4
- 需求：计算每个音素的历史正确率
- 验收标准：
  - 正确率 = 正确次数 / 总练习次数
  - 可在统计面板查看
- 依赖：DATA-001

### REQ DATA-003: 自适应推荐

- 优先级：P2
- 阶段：M4
- 需求：推荐排序考虑历史正确率（正确率低的优先）
- 验收标准：
  - 推荐列表融合 L1 难度和历史正确率
  - 已掌握音素降权
- 依赖：DATA-002, L1-002

## 13. PLAT-STATIC — 静态发布

### REQ PLAT-STATIC-001: 无后端部署

- 优先级：P0
- 阶段：M2
- 需求：MVP 可作为纯前端静态站点发布到公网
- 验收标准：
  - 构建产物不需要数据库、账号服务或自定义后端 API
  - 前端包内包含训练所需词库和 profile 数据
  - 发布文档列出 Vercel/Netlify/Cloudflare Pages/GitHub Pages 至少一种路径
- 依赖：TRAIN-CORE-005

### REQ PLAT-STATIC-002: 后端可选边界

- 优先级：P1
- 阶段：M2
- 需求：代码结构允许未来接入后端，但当前训练流程不依赖后端
- 验收标准：
  - 训练历史通过 storage/provider 抽象访问
  - 词库通过 profile/data provider 访问
  - 云端能力被标记为 M6 可选增强
- 依赖：DATA-LOCAL-002
