# IPA Spelling Master — 产品需求文档 (PRD)

## 1. 产品概述

### 1.1 产品定位
IPA Spelling Master 是一款面向美式英语学习者的国际音标(IPA)拼写训练工具。用户通过听取单词发音，在交互式音标键盘的辅助下完成 IPA 转写，从而提升语音辨识和音标掌握能力。

### 1.2 核心价值
- **听音辨音**：通过浏览器原生语音合成，建立语音与音标符号的直接映射
- **降低门槛**：内置 IPA 键盘，无需记忆或安装第三方输入法即可输入全部音标符号
- **即时反馈**：实时判断用户转写的正确性，给出正确答案对比
- **零依赖**：本地内置词库 + 浏览器 Web Speech API，无需任何外部 API，零延迟、零费用

### 1.3 产品愿景
成为英语学习者掌握美式音标的首选训练工具，覆盖从入门到精通的完整学习路径。

## 4. 功能需求详细描述

### 4.1 本地词库与难度分级 (F1)

**词库来源**：基于 COCA（Corpus of Contemporary American English，美国当代英语语料库）前 5000 词频表，经筛选去除功能词后，按词频分级。

**词库结构**：本地 TypeScript 文件内置三级难度词库，共 **4673** 词条（基础 958 / 进阶 1752 / 挑战 1963）。

| 难度 | 标签 | 词频范围 | 词汇特征 | 示例 |
|------|------|---------|---------|------|
| 基础 (Basic) | 入门 | COCA 1-1200 | 高频实义词，1-2 音节为主 | say /seɪ/, time /taɪm/, people /ˈpi:pl/ |
| 进阶 (Intermediate) | 提升 | COCA 1201-3000 | 中等频率，多音节，含常见词缀 | method /ˈmeθəd/, surface /ˈsɜ:fɪs/ |
| 挑战 (Advanced) | 精通 | COCA 3001-5000 | 低频书面语，辅音丛，多音节复杂词 | composition /ˌkɒmpəˈzɪʃn/, psychologist /saɪˈkɒlədʒɪst/ |

**随机抽取逻辑**：
1. 根据当前难度从对应词库中随机抽取 10 个不重复单词（Fisher-Yates 洗牌）
2. 每次重置时重新随机抽取，保证每轮练习的单词组合不同
3. 数百词/级确保极高的内容多样性，用户可长时间练习不重复

**触发时机**：页面首次加载 / 用户点击「New Word Set」/ 切换难度

### 4.2 浏览器原生发音 (F2)

**触发时机**：用户点击发音按钮（扬声器图标）

**处理流程**：
1. 调用 `window.speechSynthesis`
2. 创建 `SpeechSynthesisUtterance` 实例，设置：
   - `text`: 当前单词文本
   - `lang`: `'en-US'`
   - `rate`: `0.9`（稍慢语速，便于辨识）
3. 调用 `speechSynthesis.speak()` 播放

**状态管理**：
- 播放中：按钮展示音频波形动画，禁止重复点击
- 播放完成：通过 `utterance.onend` 回调恢复可点击状态
- 播放前取消任何正在进行的语音（`speechSynthesis.cancel()`）

**异常处理**：
- 浏览器不支持 SpeechSynthesis → 显示提示「您的浏览器不支持语音合成」
- 语音列表中无 en-US → 降级使用默认语音

### 4.3 IPA 交互键盘 (F3)

**布局结构**：

| 分区 | 内容 | 数量 |
|------|------|------|
| Vowels (元音) | i, ɪ, eɪ, ɛ, æ, ɑ, ɔ, oʊ, ʊ, u, ʌ, ɚ, ə, aɪ, aʊ, ɔɪ | 16 |
| Consonants (辅音) | p, b, t, d, k, ɡ, f, v, θ, ð, s, z, ʃ, ʒ, h, m, n, ŋ, l, r, j, w, tʃ, dʒ | 24 |
| Marks & Stress (标记与重音) | ˈ (主重音), ˌ (次重音), . (音节分隔), ␣ (空格) | 4 |

**交互规则**：
- 点击任意符号按钮，将其追加至输入框光标位置
- 仅当答题状态为 `neutral`（待判定）时可输入
- 答题后（correct/incorrect）键盘输入被锁定
- 键盘支持展开/折叠，由底部「Hide Keypad / Show Keypad」按钮控制

### 4.4 答案判定 (F4)

**判定逻辑**：
1. 移除用户输入和目标 IPA 两端的 `/` 斜线符号
2. 执行 `trim()` 去掉首尾空格
3. 严格字符串相等比较
4. 相等 → `correct`，得分 +1
5. 不等 → `incorrect`

**状态流转**：
```
neutral → (点击 Check Answer) → correct / incorrect
correct / incorrect → (点击 Next Challenge) → neutral（进入下一题）
```

### 4.5 进度追踪 (F5)

**展示元素**：
- 顶部进度条：以圆点/短线表示每道题的状态（已完成/未完成）
- 当前题目序号：`IPA Practice • {n} / {total}`
- 得分：累计正确数
- 准确率：`score / currentIndex * 100`（四舍五入）

### 4.6 错误提示 (F7)

回答错误时展示：
- 红色横幅：「✗ Correct: /{ipa}/」（显示标准 IPA）
- 题目区域显示：「Actual Word: **{word}**」

### 4.7 重新出题 (F8)

点击页面底部「New Word Set」→ 清空所有状态 → 从当前难度词库重新随机抽取 10 题 → 进度归零

### 4.8 难度切换 (F9)

难度选择器位于 Header 区域，三个选项：基础 / 进阶 / 挑战。切换难度时自动重置题目和分数。

### 4.9 会话结束

完成全部 10 题后弹出完成面板，显示得分和准确率，用户可选择「再来一轮」或「切换难度」。

## 5. 非功能需求

### 5.1 性能

| 指标 | 目标值 |
|------|--------|
| 首屏加载时间 | < 1.5s |
| 出题抽选延迟 | < 50ms（纯内存操作） |
| 发音触发延迟 | < 200ms（浏览器本地） |
| 页面交互响应 | < 100ms |

### 5.2 可用性
- 界面适配桌面端（后续版本适配移动端）
- 所有可点击元素提供 hover/active 视觉反馈
- 按钮禁用状态有明确视觉区分（opacity: 0.3）
- 可纯静态部署，无需 Node.js 服务端

### 5.3 可靠性
- 零外部 API 依赖，不受网络波动影响
- Web Speech API 不可用时优雅降级，显示提示信息
- 输入验证：无输入时「Check Answer」按钮禁用

### 5.4 可维护性
- 前端使用 TypeScript 提供类型安全
- 组件化设计（IPAKeypad 独立组件，词库独立模块）
- 词库与业务逻辑分离，便于扩充
- 无后端代码，纯前端 SPA

### 5.5 安全性
- 无服务端，无 API Key 泄露风险
- 无用户数据采集或网络传输
- 所有数据存储在浏览器内存中
